Sends events to the Amazon CloudWatch HTTP Log Collector.

```tql
to_amazon_cloudwatch_hlc group:string, stream:string, [message=string|blob,
  timestamp=time, batch_timeout=duration, batch_size=int, token=secret,
  endpoint=string, aws_region=string]
```

## Description

The `to_amazon_cloudwatch_hlc` operator sends events to the CloudWatch HTTP Log
Collector with bearer-token authentication.

Use <Op>to_amazon_cloudwatch</Op> to write with the AWS `PutLogEvents` API.

### `group: string`

Name of the CloudWatch log group.

### `stream: string`

Name of the CloudWatch log stream.

### `message = string|blob (optional)`

Expression that computes the log message for each event.

Defaults to the incoming event serialized as NDJSON.

### `timestamp = time (optional)`

Expression that computes the CloudWatch event timestamp.

Defaults to a `timestamp` field when present. Otherwise, the operator uses the
processing time.

### `batch_timeout = duration (optional)`

Maximum time to keep events in a batch before flushing them.

Defaults to `1s`.

### `batch_size = int (optional)`

Maximum number of events per batch.

Defaults to `1000`.

### `token = secret`

Bearer token for the CloudWatch HTTP Log Collector.

### `endpoint = string (optional)`

Base URL of the CloudWatch HTTP Log Collector endpoint.

Defaults to `https://logs.<region>.amazonaws.com`, where `<region>` comes from
`aws_region`. Set `endpoint` to use a custom collector URL.

The operator sends requests to
`/services/collector/event?logGroup=<group>&logStream=<stream>` with
URL-encoded group and stream names.

### `aws_region = string (optional)`

AWS region for the default CloudWatch HTTP Log Collector endpoint.

This option is required when `endpoint` is omitted.

## Batching

The operator sends batches as JSON requests. Events are skipped with a warning
when the JSON request envelope would exceed the 1 MiB request limit.

## Examples

### Send events to the HTTP Log Collector

```tql
from {message: "hello"}
to_amazon_cloudwatch_hlc "/tenzir/events", "default",
  message=message,
  token=secret("cloudwatch-hlc-token"),
  aws_region="us-east-1"
```

### Use an explicit endpoint

```tql
from {message: "hello"}
to_amazon_cloudwatch_hlc "/tenzir/events", "default",
  message=message,
  token=secret("cloudwatch-hlc-token"),
  endpoint="https://logs.us-east-1.amazonaws.com"
```
