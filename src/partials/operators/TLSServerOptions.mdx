### `tls_client_ca = string (optional)`

Path to a CA certificate file for validating client certificates (server mode only).
When set along with `tls_require_client_cert`, the server will verify client
certificates against this CA.

Defaults to the configuration value `tenzir.operator-tls.tls-client-ca` or no
certificate otherwise.

### `tls_require_client_cert = bool (optional)`

Require clients to present valid certificates (server mode only). When enabled,
clients MUST present a certificate signed by the CA specified in `tls_client_ca`,
otherwise the connection will be rejected. This enables mutual TLS (mTLS)
authentication.

Defaults to the configuration value `tenzir.operator-tls.tls-require-client-cert`
or `false` if it is not set.
